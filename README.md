# Резидентный монитор регистров с тройной буферизацией

Данный проект представляет собой резидентную программу (TSR) для DOS, которая при нажатии левой клавиши `Ctrl` выводит на экран рамку с текущими значениями всех основных регистров процессора (AX, BX, CX, DX, SI, DI, BP, SP, DS, ES, SS, CS, IP). Значения регистров «замораживаются» в момент нажатия и не меняются до отпускания клавиши. Во время отображения рамки фон под ней непрерывно обновляется, отслеживая изменения на экране (ввод пользователя, вывод других программ) через обработчик прерывания таймера (int 8). При отпускании `Ctrl` исходное содержимое экрана восстанавливается.

## Принцип работы

### 1. Захват прерываний
- **int 9 (клавиатура)** - отслеживает нажатие и отпускание левого `Ctrl`.
- **int 8 (таймер)** - вызывается примерно 18.2 раза в секунду и используется для мониторинга изменений видеопамяти.

### 2. Буферизация (тройная логика)
Для обеспечения корректного восстановления экрана и неизменности рамки используются два буфера:

- **Буфер фона (`BUFFER_SCREEN`)** - точная копия видеопамяти (80x25 символов, 4000 байт) на момент последнего обновления фона.
- **Буфер рамки (`BUFFER_FRAME`)** - статичное изображение рамки с регистрами, сформированное один раз при нажатии `Ctrl`.
- **Видеопамять** - буфер с изображением из фона и рамки, которое видит пользователь.

При активной рамке каждый тик таймера происходит сравнение текущего содержимого видеопамяти с этими буферами:
- Если изменилась область **вне рамки**, изменения копируются в буфер фона.
- Если изменилась область **внутри рамки**, изменения также копируются в буфер фона (чтобы фон под рамкой оставался актуальным), и устанавливается флаг необходимости перерисовки рамки.
- Если флаг перерисовки установлен, рамка заново выводится на экран из буфера рамки, восстанавливая свой неизменный вид поверх обновлённого фона.

### 3. Сохранение регистров через стек
В момент нажатия `Ctrl` все регистры (включая IP, полученный через `call get_ip`) последовательно заталкиваются в стек. Процедура `MakeFrameBuffer` читает их напрямую из стека, используя `bp` со смещением, и формирует буфер рамки. Это позволяет избежать использования отдельного массива и гарантирует, что значения регистров будут именно теми, что были в момент нажатия.

### 4. Гибкая настройка положения рамки
Положение рамки задаётся константами `DEF_STR` (строка) и `DEF_COL` (столбец). Все проверки границ и вычисления смещений используют эти константы, поэтому рамку можно легко переместить в любое место экрана, изменив всего два значения.

## Основные процедуры

| Процедура | Назначение |
|-----------|------------|
| `New09h`  | Обработчик клавиатуры: реагирует на левый `Ctrl`. |
| `New08h`  | Обработчик таймера: обновляет фон и перерисовывает рамку при необходимости. |
| `SaveScreen` / `RestoreScreen` | Копирование экрана в/из буфера фона. |
| `MakeFrameBuffer` | Читает регистры из стека и формирует буфер рамки. |
| `PrintFrameBuffer` | Выводит буфер рамки на экран в заданную позицию. |

## Использование

1. Запуск:

    * Просто выполните resident.com - программа установит себя в память и завершится (останется резидентом).

    * После этого в любой момент нажмите левый Ctrl - появится рамка с регистрами.

    * Для выхода из режима просмотра отпустите Ctrl - экран восстановится.

    *Чтобы выгрузить резидент, потребуется перезагрузка или специальная утилита (в данной версии не реализовано).

2. **Настройка положения рамки:**

    В исходном коде измените константы `DEF_STR` (строка, 0-24) и `DEF_COL` (столбец, 0-79) и пересоберите программу.

## Ограничения
* Не реализовано автоматическое выгрузка из памяти (только перезагрузка).

* Не обрабатываются другие видеорежимы (только стандартный 80x25 цветной текст).

* При очень интенсивном изменении экрана возможна небольшая задержка обновления фона (до 1/18 секунды).
